# Дизайн: простий рекордер екрана для Android

**Дата:** 2026-06-07
**Цільовий пристрій:** Samsung на Android 14
**Мета:** дуже простий додаток для запису відео + звуку з екрана, який дитина зможе самостійно запускати й зупиняти.

## Мотивація

На Samsung дитини немає вбудованого рекордера екрана. Потрібен власний мінімальний додаток: одна кнопка «Почати», зупинка зі шторки сповіщень, готове відео в Галереї.

## Вимоги

### Функціональні
- Запис відео з екрана (роздільність як у екрана, авто).
- Запис **системного звуку** завжди.
- Запис **мікрофона** опціонально — перемикач, можна вимкнути.
- Системний звук + мікрофон **змішуються в одну аудіодоріжку**.
- Старт — кнопкою в додатку. Стоп — кнопкою у сповіщенні в шторці.
- Готовий файл MP4 з'являється в Галереї.

### Нефункціональні
- Максимальна простота для дитини: один екран, мінімум елементів.
- Без налаштувань якості — усе автоматично (єдиний перемикач: мікрофон).
- Тільки стандартні Android API, без сторонніх бібліотек.

### Поза межами (YAGNI)
- Налаштування бітрейту / роздільності / FPS.
- Плаваюча кнопка, керування жестами.
- Редагування / обрізка відео.
- Хмарне завантаження, шерінг.
- Пауза/відновлення запису.

## Технологічний стек
- Нативний Kotlin, Android Studio.
- Один екран на Jetpack Compose.
- `minSdk` = 29 (потрібно для AudioPlaybackCapture), `targetSdk` = 34 (Android 14).
- Жодних залежностей поза стандартним Android SDK.

## Архітектура

Три компоненти з чіткими межами:

### 1. UI-екран (`MainActivity` + Compose)
- Перемикач «Мікрофон: вкл/викл» (стан зберігається).
- Велика кнопка «Почати запис».
- При натисканні: запит дозволу `RECORD_AUDIO` (якщо ще нема) і запит `POST_NOTIFICATIONS` (Android 13+), далі системне вікно `MediaProjection`.
- Після згоди — запускає фоновий сервіс і ховається/показує стан «Йде запис».
- **Залежить від:** дозволів, `MediaProjectionManager`.

### 2. Фоновий сервіс запису (`RecordingService`)
- `foreground service` типу `mediaProjection` (обов'язково для Android 14).
- Показує стале сповіщення з дією **«Зупинити»**.
- Тримає `MediaProjection`, керує життєвим циклом конвеєра кодування (старт/стоп).
- На стоп: коректно закриває кодеки/мукер, зберігає файл, прибирає сповіщення.
- **Залежить від:** `MediaProjection` (передається з Activity), конвеєра кодування.

### 3. Конвеєр кодування (`ScreenEncoder`)
Серце додатка. Інкапсулює всю роботу з медіа за чітким інтерфейсом (`start()`, `stop()`).

**Відеошлях:**
- `MediaProjection.createVirtualDisplay()` → рендерить екран у вхідний `Surface` відеокодека.
- `MediaCodec` (H.264/AVC), вхід — `Surface`.
- Вихідні буфери → `MediaMuxer` (відеодоріжка).

**Аудіошлях:**
- `AudioRecord` із `AudioPlaybackCaptureConfiguration` (системний звук).
- Якщо мікрофон увімкнено — другий `AudioRecord` із мікрофона.
- PCM-буфери **змішуються вручну** (просте сумування семплів із захистом від переповнення).
- Змішаний PCM → `MediaCodec` (AAC) → `MediaMuxer` (аудіодоріжка).
- Якщо мікрофон вимкнено — лише системний звук, без мікшування.

**Контейнер:**
- `MediaMuxer` збирає відео- й аудіодоріжки в один **MP4**.

### Потік даних
```
Екран ──► VirtualDisplay ──► [Surface] ──► VideoCodec(H264) ──┐
                                                              ├──► MediaMuxer ──► MP4
Система ─► AudioRecord(playback) ─┐                           │
Мікрофон ► AudioRecord(mic) ──────┴─► PCM mix ─► AudioCodec(AAC)┘   (mic — опціонально)
```

## Збереження
- MP4 пишеться в спільну колекцію `MediaStore.Video` → папка `Movies/ScreenRec`.
- Завдяки MediaStore файл одразу індексується Галереєю; дозволи на зовнішнє сховище не потрібні (scoped storage).
- Імена файлів: `ScreenRec_<timestamp>.mp4`.

## Дозволи
- `RECORD_AUDIO` — мікрофон і захоплення системного звуку.
- `FOREGROUND_SERVICE` + `FOREGROUND_SERVICE_MEDIA_PROJECTION`.
- `POST_NOTIFICATIONS` (Android 13+) — для сповіщення сервісу.
- Згода `MediaProjection` — системне вікно при кожному старті (прибрати не можна, обмеження ОС).

## Обробка помилок
- Відмова в дозволі мікрофона → запис без мікрофона (повідомлення), не падати.
- Відмова в `MediaProjection` → нічого не запускати, повернутись на екран.
- Помилка кодека / нема місця → зупинити, прибрати неповний файл, показати тост.
- Дзвінок / інше захоплення аудіо → система може перервати; коректно завершити файл.

## Відоме обмеження
`AudioPlaybackCapture` захоплює звук лише з додатків, що це дозволяють (за замовчуванням — більшість ігор/відео). Захищений контент (Netflix тощо, інколи Spotify) блокує захоплення свого звуку — там вийде відео без їхнього звуку. Це обмеження Android, обійти не можна.

## Тестування
- Перевірка конвеєра кодування на коректних/некоректних послідовностях буферів (юніт-логіка мікшування PCM).
- Ручна перевірка на пристрої: запис гри зі звуком + мікрофоном; запис із вимкненим мікрофоном; перевірка появи файлу в Галереї; відтворення MP4.
- Перевірка сценаріїв дозволів (відмова мікрофона, відмова MediaProjection).

## Встановлення на телефон дитини
Збірка APK у Android Studio → встановлення через USB-кабель (adb) або копіюванням APK-файлу на телефон. Деталі — на етапі плану.
